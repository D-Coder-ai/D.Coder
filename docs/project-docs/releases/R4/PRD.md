# Release R4 (GA) — Product Requirements Document (PRD)

Platform: D.Coder (reusable foundation for future platforms)

Primary Tenant (reference): Deloitte USI IGS

Status: Planning — Documentation-First

## 1. Goals
- General Availability with hardened security, compliance, and operations.
- Support broader customer base and platform reuse beyond D.Coder branding.

## 2. Scope (Functional)
- Mature integrations marketplace UX for plugins (discovery, install, configure, updates).
- Advanced quotas/budgets: automatic policy actions (throttle/deny) and approvals.
- Optional automatic provider failover with clear audit and cost ceilings.

## 3. Scope (Non‑Functional)
- SLOs: Availability 99.9%+, latency SLOs per surface, capacity plans.
- DR: Validated RPO/RTO targets; regular game‑days.
- Compliance: SOC2 audit readiness with evidence; optional HIPAA readiness.

## 4. Acceptance Criteria
- Successful multi‑tenant production pilots with plugins enabled per tenant.
- Compliance evidence pack assembled; DR tests executed.
- Documentation finalized for GA: PRD, addendum, ops runbooks.
